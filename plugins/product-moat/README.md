# Product Moat

A Claude Code plugin that runs an interactive interview to pressure-test your product's defensibility in the age of AI.

```
                    ┌─────────────────────┐
                    │    COMPOUND          │
                    │    INTELLIGENCE      │

                    learns from usage
                               │
              ┌────────────────┼────────────────┐
              ▼                                  ▼
   ┌─────────────────┐              ┌─────────────────────┐
   │   SERVICE AS     │◄────────────►│   PRESCRIPTIVE      │
   │   SOFTWARE       │   feeds &    │   DATA              │
   │                  │   enables    │                     │
   └─────────────────┘              └─────────────────────┘
```

## The Three Moats

Most AI products have no real moat. This tool helps you find out if yours does.

- **Compound Intelligence** — Does your product get smarter from usage, not just from training data?
- **Service as Software** — Do you deliver outcomes, or just tools?
- **Prescriptive Data** — Do you tell users what to do, or just what happened?

## Install

```bash
claude plugin install https://github.com/YOUR_USERNAME/product-moat
```

## Usage

```
/product-moat:moat-interview [your product name]
```

The interview runs in four stages:

1. **Context** — Brief setup about your product and stage
2. **Deep dive** — Section-by-section assessment across all three moats
3. **Honest check** — The uncomfortable questions
4. **Scorecard** — Strengths, gaps, and concrete 30-day actions

## What to expect

This is not a checklist. It's a Socratic sparring session. Expect to be challenged on vague answers. The goal is to find the gaps before the market does.
